## 1. 产品定位

**顺时养生**是一个动态决策系统，基于"节气×地域×体质"三层参数化推理引擎，为用户提供个性化节气养生方案。核心差异化：

- 同一节气，不同城市输出不同方案（地域适配）
- 同一节气+城市，不同体质输出不同方案（体质适配）
- 每天根据天气自动调整食谱（每日不重复）

## 4. 功能定义

### 4.1 P0 核心功能

#### 功能1：节气养生方案生成

| 属性 | 说明 |
|------|------|
| **输入** | 节气（自动获取）+ 地域（自动定位+手动选择）+ 体质（用户自选或测评结果） |
| **输出** | 三维方案：健康建议 + 起居建议 + 饮食方案 |
| **规则** | 三层参数化推理引擎（CM1不变层→CM2地域适配→CM3体质适配→CM4三维合成） |
| **更新频率** | 用户触发（点击刷新）+ 自动（每日06:00） |
| **UI** | 卡片式展示，每个维度一个卡片，支持展开/收起 |

#### 功能2：体质测评问卷

| 属性 | 说明 |
|------|------|
| **输入** | 8题是/否选择题 |
| **输出** | 体质类型（九种之一或混合体）+ 体质得分 |
| **规则** | 九种体质匹配算法（加权评分） |
| **UI** | 步骤式问卷，每题一屏，进度条显示 |
| **存储** | 用户历史测评结果，支持重新测评 |

#### 功能3：每日天气驱动食谱

| 属性 | 说明 |
|------|------|
| **输入** | 节气 + 地域 + 当日天气（和风天气API） + 体质 |
| **输出** | 一日三餐食谱 + 养生汤品 |
| **规则** | 食材池轮转（候递进×天气修正×日期种子） |
| **更新频率** | 每日自动更新（06:00） |
| **UI** | 卡片式展示，每餐一个卡片，含食材清单+简要做法 |

#### 功能4：每日花草茶推荐

| 属性 | 说明 |
|------|------|
| **输入** | 节气 + 地域 + 当日天气 + 体质（与食谱共用） |
| **输出** | 今日花草茶配方（主配方 + 备选配方）+ 冲泡方法 |
| **规则** | 三层参数化推理引擎（CM1不变层→CM2地域适配→CM3体质适配→天气修正），完全复用现有引擎 |
| **更新频率** | 每日自动更新（06:00） |
| **UI** | 独立卡片，展示配方名称、食材清单、冲泡方法、体质适配说明 |
| **数据源** | 花草茶配方池（herbal_tea_pool），结构与 recipe_pool 一致 |

## 6. 数据模型

### 6.1 核心数据表

| 表名 | 条目数 | 说明 |
|------|--------|------|
| `solar_term_registry` | 24条 | 节气不变层（天文/气候/阴阳/脏腑/方向/脆弱点） |
| `city_climate` | 300条+ | 城市气候数据（气候类型/温度/湿度/偏差） |
| `constitution_adaptation` | 9×24条 | 九种体质×节气脆弱点映射 |
| `weather_correction` | 10条 | 天气类型×食材权重修正系数 |
| `recipe_pool` | 24×9条 | 节气+体质→食材池（JSON字段） |
| `user_profile` | 动态 | 用户体质/地域/历史测评 |
| `recipe_history` | 动态 | 用户历史食谱记录 |

### 6.2 节气不变层表结构（solar_term_registry）

```sql
CREATE TABLE solar_term_registry (
  id INT PRIMARY KEY AUTO_INCREMENT,
  term_name VARCHAR(10) NOT NULL,         -- 节气名称
  date_range VARCHAR(50) NOT NULL,         -- 日期范围
  solar_longitude VARCHAR(20) NOT NULL,    -- 太阳黄经
  climate_pattern VARCHAR(200) NOT NULL,   -- 气候规律
  yinyang_attribute VARCHAR(50) NOT NULL,  -- 阴阳属性
  tcm_organ VARCHAR(50) NOT NULL,          -- 中医当令脏腑
  wellness_direction VARCHAR(100) NOT NULL,-- 养生方向
  vulnerability_points TEXT NOT NULL,      -- 脆弱点
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6.3 体质适配表结构（constitution_adaptation）

```sql
CREATE TABLE constitution_adaptation (
  id INT PRIMARY KEY AUTO_INCREMENT,
  constitution_type VARCHAR(20) NOT NULL,  -- 体质类型
  solar_term VARCHAR(10) NOT NULL,         -- 节气
  city VARCHAR(20) DEFAULT '通用',         -- 城市（地域适配）
  vulnerability_points TEXT NOT NULL,      -- 脆弱点
  direction_override VARCHAR(100),         -- 方向修正（如有）
  intensity_modifier VARCHAR(100),         -- 程度修正
  focus_shift VARCHAR(100),                -- 侧重点偏移
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 7. API设计

### 7.1 核心API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/solar-term/current` | 获取当前节气信息 |
| GET | `/api/wellness-plan` | 获取节气养生方案（节气+地域+体质） |
| GET | `/api/recipe/daily` | 获取当日食谱（节气+地域+体质+天气） |
| GET | `/api/herbal-tea/daily` | 获取当日花草茶（节气+地域+体质+天气） |
| POST | `/api/constitution/assess` | 体质测评提交 |
| GET | `/api/constitution/{userId}` | 获取用户体质信息 |
| GET | `/api/weather/{city}` | 获取当日天气 |
| GET | `/api/solar-term/countdown` | 获取节气倒计时 |
| GET | `/api/ingredients/{term}` | 获取节气食材推荐 |

## 8. 用户流程

### 8.1 首次使用

```
打开小程序 → 自动定位 → 显示今日节气卡片 → 引导体质测评（8题） → 生成首份方案 → 展示方案
```

### 8.2 每日使用

```
打开小程序 → 显示今日节气卡片 → 显示方案（基于当日天气自动更新） → 查看详细/收藏
```

### 8.3 节气转换

```
节气前3天 → 推送提醒 → 用户点击推送 → 进入节气详情页 → 查看新节气方案
```
