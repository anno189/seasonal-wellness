## 1. 产品概述

### 1.1 愿景

顺时生活是一个基于中医"天人合一"理念的个人化节气养生助手，通过"节气 × 地域 × 体质"三层参数化推理引擎，为每位用户生成每日动态调整的养生方案——包括饮食、起居、花草茶与健康建议。

### 1.2 核心定位

| 维度 | 定位 |
|------|------|
| **产品形态** | 微信小程序（主力）+ 网站 PWA（内容引流） |
| **核心差异** | 同一节气，不同城市 → 不同方案；同一城市，不同体质 → 不同方案；每天根据天气 → 不同食谱 |
| **技术路线** | 规则引擎驱动，无需大模型，成本可控、结果稳定可解释 |
| **发展阶段** | 技术验证版 → 正式 MVP 上线（v2.0） |

### 1.3 一句话价值主张

> 顺应节气、因人而异——每天打开小程序，获得专属于你的今日养生方案。

## 2. 市场分析

### 2.1 市场背景

- **中医养生赛道持续增长**：国潮文化兴起，年轻用户对中医养生接受度快速提升
- **个性化健康需求升级**：用户不再满足于通用养生知识，期望获得"为我定制"的建议
- **节气内容天然适配小程序**：24节气是中国文化共识，内容天然具有传播性和时效性，适合碎片化消费场景
- **现有竞品多为内容型**：市面上以节气养生为主题的产品多为知识科普类，缺乏真正的个性化方案生成能力

### 2.2 目标用户画像

| 用户类型 | 占比 | 年龄段 | 核心需求 | 典型使用场景 |
|----------|------|--------|----------|-------------|
| **养生爱好者** | 55% | 35-55岁 | 节气养生知识，个性化饮食建议 | 厨房查食谱、清晨查看当日建议 |
| **中医文化爱好者** | 25% | 30-60岁 | 节气文化、中医体质调理 | 睡前回顾当日方案、节气节点查看 |
| **健康焦虑型** | 20% | 25-45岁 | 快速获得权威养生建议 | 通勤中碎片化查阅 |

### 2.3 用户旅程

```
认知：网站 SEO 内容 / 朋友推荐 → 搜索"节气养生"
  ↓
转化：打开小程序 → 自动定位城市 → 显示今日节气卡片
  ↓
激活：引导完成 8 题体质测评 → 生成首份个性化方案
  ↓
留存：每日打开查看当日方案（天气驱动，每日不重复）
  ↓
增长：分享花草茶配方 / 节气卡片给朋友
```

## 3. 功能规划

### 3.1 P0 核心功能（v2.0 MVP 必须交付）

#### 功能 1：节气养生方案生成

| 属性 | 说明 |
|------|------|
| **输入** | 当前节气（自动获取）+ 城市（自动定位 + 手动选择）+ 体质（测评结果或默认平和质） |
| **输出** | 三维方案：健康建议 + 起居建议 + 饮食方案 |
| **推理引擎** | 四层参数化推理：CM1 节气不变层 → CM2 地域适配 → CM3 体质适配 → CM4 三维合成 |
| **更新频率** | 用户触发（下拉刷新）+ 每日 06:00 自动更新 |
| **UI 形式** | 首页卡片式展示，点击展开查看详情页 |
| **状态说明** | v1.0 已验证，v2.0 完善数据覆盖与异常处理 |

**方案输出结构：**

```
健康维度
├── 今日高风险（如中暑、空调病）
├── 预防措施（具体可执行）
└── 体质脆弱点分析（个性化说明）

起居维度
├── 睡眠建议（入睡时间 + 午睡建议）
├── 运动建议（时间 + 推荐运动）
└── 情绪调节

饮食维度
├── 早餐食谱（名称 + 食材清单）
├── 午餐食谱（名称 + 食材清单）
├── 晚餐食谱（名称 + 食材清单）
└── 养生汤品（名称 + 备注）
```

#### 功能 2：体质测评问卷

| 属性 | 说明 |
|------|------|
| **输入** | 8 题调研式问答题（基于中华中医药学会《中医体质分类与评定》标准） |
| **输出** | 体质类型（九种之一）+ 体质特征 + 养生原则 |
| **覆盖体质** | 平和质、气虚质、阳虚质、阴虚质、痰湿质、湿热质、血瘀质、气郁质、特禀质 |
| **匹配算法** | 加权评分法，支持单一体质与混合体质判定 |
| **UI 形式** | 步骤式问卷，每题一屏，含进度条 |
| **数据存储** | 本地存储（微信 Storage），支持随时重新测评 |
| **状态说明** | v1.0 已验证，v2.0 优化问卷交互与结果展示 |

**问卷题目示例：**

| 编号 | 题目 | 目标体质（权重） |
|------|------|-----------------|

| 2 | 您比一般人怕冷，手脚偏凉，喜欢热饮食吗？ | 阳虚质(3)、气虚质(1) |
| 3 | 您经常感到口干、手心脚心发热、夜间盗汗或失眠多梦吗？ | 阴虚质(3) |
| 4 | 您体型偏胖，腹部肥满松软，平时痰多，容易胸闷吗？ | 痰湿质(3)、湿热质(1) |
| 5 | 您面部油光、易生痤疮或疖肿，口苦口干、大便黏滞不爽吗？ | 湿热质(3)、痰湿质(1) |
| 6 | 您皮肤容易出现瘀斑，口唇颜色偏暗，女性月经多血块吗？ | 血瘀质(3) |
| 7 | 您平时情绪抑郁、烦躁易怒、容易胸闷叹息或睡眠不安吗？ | 气郁质(3) |
| 8 | 您容易过敏（如鼻炎、哮喘、荨麻疹），对花粉、药物或某些食物敏感吗？ | 特禀质(3) |

### 5.4 规则引擎设计（核心）

**四层参数化推理引擎：**

```javascript
// 方案生成流程（伪代码）

class WellnessPlanService {
  async generatePlan({ term, city, constitution, date }) {
    // CM1：节气不变层 — 方向锚定
    const invariance = InvarianceLayer.getInvariance(term);
    const hou = InvarianceLayer.getCurrentHou(term, dayInTerm);

    // CM2：地域气候适配 — 程度修正
    const regional = RegionalAdaptation.getAdaptation(city, term);

    // CM3：体质适配 — 侧重点偏移
    const constitutionData = ConstitutionAdaptation.getAdaptation(
      constitution, term, city
    );

    // CM4：三维方案合成
    const plan = PlanSynthesis.synthesize({
      invariance,
      regional,
      constitutionData,
      hou,
    });

    return plan;
  }
}
```

**食材池轮转算法：**

```javascript
// 食谱生成流程（伪代码）

class RecipeService {
  async getDailyRecipe({ term, city, constitution, weather, date }) {
    const foodPool = this.getFoodPool(term, city, constitution);
    const weatherWeights = WeatherCorrection.getWeights(weather);
    const seed = this.generateSeed(date);

    const breakfast = this.weightedRandom(foodPool, weatherWeights, seed);
    const lunch = this.weightedRandom(foodPool, weatherWeights, seed + 1);
    const dinner = this.weightedRandom(foodPool, weatherWeights, seed + 2);

    return { breakfast, lunch, dinner };
  }
}
```

**花草茶生成（共享引擎）：**

```javascript
// 花草茶生成流程（伪代码）

class HerbalTeaService {
  async getDailyTea({ term, city, constitution, weather, date }) {
    const teaPool = this.getTeaPool(term, city, constitution);
    const weatherWeights = WeatherCorrection.getWeights(weather);
    const seed = this.generateSeed(date);

    const primary = this.weightedRandom(teaPool, weatherWeights, seed);
    const alternative = this.weightedRandom(teaPool, weatherWeights, seed + 1);

    return { primary, alternative };
  }
}
```

**候递进规则：**

| 候 | 天数 | 强度系数 | 食材池策略 |
|----|------|---------|-----------|
| 初候 | 第 1-5 天 | 0.8（温和） | 以当前节气食材为主 |
| 中候 | 第 6-10 天 | 1.0（标准） | 当前节气食材 + 适度加强 |
| 末候 | 第 11-15 天 | 1.2（加强） | 当前节气 + 下一节气过渡食材 |

## 7. 商业化方案

### 7.1 整体策略

分两阶段推进：**第一阶段以流量获取和内容积累为主，第二阶段探索变现。**

```
阶段一（0-6 个月）：用户增长 + 数据积累
  → 免费使用全部功能
  → 内容 SEO 引流（网站 PWA）
  → 微信生态裂变（分享花草茶/节气卡片）
  → 收集用户行为数据，优化推荐算法

阶段二（6-12 个月）：变现探索
  → 广告变现（流量稳定后接入）
  → 会员订阅（核心用户付费意愿验证后上线）
```

### 7.2 广告变现方案

| 广告类型 | 投放位置 | 预期 CPM | 备注 |
|---------|---------|---------|------|
| **Banner 广告** | 首页底部 / 详情页底部 | ¥10-30/千次展示 | 非侵入式，不影响核心体验 |
| **原生信息流广告** | 方案详情页底部 | ¥20-50/千次展示 | 与内容风格一致 |
| **激励视频广告** | 查看高级花草茶配方 | ¥50-100/次观看 | 用户主动选择，体验好 |

**接入时机**：日活用户稳定在 1,000+ 后再接入，避免早期干扰用户体验。

### 7.3 会员订阅方案

| 会员等级 | 月费（预估） | 权益 |
|---------|------------|------|
| **免费版** | ¥0 | 全部基础功能（方案/食谱/花草茶/测评） |
| **会员版** | ¥12-18/月 | 去广告 + 高级花草茶配方库 + 节气深度调理方案 + 体质变化追踪 |

**会员功能设计原则**：
- 核心功能（方案生成、体质测评）始终保持免费
- 付费功能聚焦于"深度"而非"基础"——高级配方、深度分析、历史追踪
- 提供 7 天免费试用，降低转化门槛

### 7.4 其他变现可能（远期）

| 方向 | 说明 | 时机 |
|------|------|------|
| **电商导流** | 推荐节气食材、花草茶原料，对接电商平台（CPS 分成） | 用户量 5 万+ |
| **品牌合作** | 与养生食品/茶饮品牌合作内容 | 品牌知名度建立后 |
| **API 输出** | 将规则引擎能力开放给其他健康类应用 | 引擎成熟后 |
