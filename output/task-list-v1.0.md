# 顺时养生 MVP 开发任务清单

> 日期：2026-07-20
> 依据：PRD v1.0 + ADR v1.0
> 方法论：agile-delivery-methodology + rapid-prototyping-methodology
> 目标：从零到可上线的小程序，约3个月

---

## 一、规格分析（agile-delivery-methodology）

### 1.1 忠实引用 P0 功能

| 功能 | PRD出处 | 复杂度 |
|------|---------|--------|
| 节气养生方案生成 | §4.1 功能1 | 中 |
| 体质测评问卷 | §4.1 功能2 | 低 |
| 每日天气驱动食谱 | §4.1 功能3 | 中 |
| 每日花草茶推荐 | §4.1 功能4 | 低 |

### 1.2 规格缺口识别（待确认）

| 缺口 | 说明 | 影响 |
|------|------|------|
| 24节气完整知识数据 | 需填充所有节气数据 | 阻塞后端开发 |
| 3城市气候数据 | 需填充北京/上海/广州气候数据 | 阻塞地域适配 |
| 24×9体质适配表 | 需填充所有节气+体质组合 | 阻塞规则引擎 |
| 24×9食材池 | 需填充所有节气+体质食材池 | 阻塞食谱服务 |
| 24×9花草茶池 | 需填充所有节气+体质花草茶池 | 阻塞花草茶服务 |
| 小程序命名 | 未确定 | 阻塞小程序注册 |
| 免责声明文本 | 未确定 | 阻塞方案输出 |

### 1.3 技术栈提取

- 前端：Taro v3.x + Vant Weapp
- 后端：Express.js + MySQL 8.0
- 天气API：和风天气免费版
- 部署：腾讯云轻量服务器 2C2G

---

## 二、任务拆解（10个任务，每个30-60分钟）

### Phase 1：环境搭建（约2周）

#### TASK-01：项目脚手架搭建

| 字段 | 内容 |
|------|------|
| **描述** | 使用 Taro 脚手架创建小程序项目，初始化后端 Express 项目，配置 MySQL 连接 |
| **验收标准** | 1. `taro init` 成功创建项目 2. `npm start` 本地启动小程序 3. 后端 Express 启动成功 4. MySQL 连接成功 5. `GET /api/health` 返回 `{"status": "ok"}` |
| **涉及文件** | `frontend/` (Taro项目), `backend/src/app.js`, `backend/src/config/db.js` |
| **前置** | 无 |
| **规格出处** | PRD §6 数据模型, ADR-002/003/004 |

#### TASK-02：数据库表结构创建

| 字段 | 内容 |
|------|------|
| **描述** | 根据 PRD §6 数据模型创建7张数据库表（含花草茶池表），创建索引 |
| **验收标准** | 1. 7张表全部创建成功 2. 每条表有注释 3. 主键索引创建成功 4. `solar_term_registry` 有24条测试数据 5. `city_climate` 有3条测试数据（北京/上海/广州） |
| **涉及文件** | `backend/src/models/*.js`, `backend/src/config/schema.sql` |
| **前置** | TASK-01 |
| **规格出处** | PRD §6.1-6.3, ADR-004 |

#### TASK-03：节气不变层数据填充（24节气）

| 字段 | 内容 |
|------|------|
| **描述** | 填充24节气的不变层数据（天文/气候/阴阳/脏腑/方向/脆弱点），基于 `kd-full.yaml` 蒸馏产物 |
| **验收标准** | 1. 24条数据全部填充 2. 每条包含6个维度 3. 数据与 `kd-full.yaml` 一致 4. 查询响应时间 ≤100ms |
| **涉及文件** | `backend/src/config/solar_terms.json` |
| **前置** | TASK-02 |
| **规格出处** | PRD §6.1, `knowledge-templates/kd-full.yaml` |

#### TASK-04：地域气候数据填充（3城市）

| 字段 | 内容 |
|------|------|
| **描述** | 填充北京/上海/广州的气候数据（气候类型/温度/湿度/偏差） |
| **验收标准** | 1. 3个城市数据全部填充 2. 每个城市包含气候类型、冬夏温度、湿度范围、偏差方向 3. 与真实气候数据一致 |
| **涉及文件** | `backend/src/config/cities.json` |
| **前置** | TASK-02 |
| **规格出处** | PRD §6.1, ADR-006 |

### Phase 2：核心规则引擎（约2周）

#### TASK-05：规则引擎 CM1-CM4 核心逻辑

| 字段 | 内容 |
|------|------|
| **描述** | 实现三层参数化推理引擎（CM1不变层→CM2地域适配→CM3体质适配→CM4三维合成），输出方案卡片数据 |
| **验收标准** | 1. 输入：节气+城市+体质，输出：三维方案（健康/起居/饮食） 2. CM1正确提取节气不变层 3. CM2正确应用地域修正 4. CM3正确应用体质适配 5. CM4正确合成三维方案 6. 响应时间 ≤1秒 |
| **涉及文件** | `backend/src/engines/InvarianceLayer.js`, `backend/src/engines/RegionalAdaptation.js`, `backend/src/engines/ConstitutionAdaptation.js`, `backend/src/engines/PlanSynthesis.js` |
| **前置** | TASK-03, TASK-04 |
| **规格出处** | PRD §4.1 功能1, 技术架构 §4.1 |

#### TASK-06：体质适配表填充（9×24）

| 字段 | 内容 |
|------|------|
| **描述** | 填充九种体质×24节气的适配数据（脆弱点/方向修正/程度修正/侧重点偏移） |
| **验收标准** | 1. 216条数据全部填充（9体质×24节气） 2. 每条包含脆弱点、方向修正、程度修正、侧重点偏移 3. 数据与 `kd-full.yaml` 一致 |
| **涉及文件** | `backend/src/config/constitutions.json` |
| **前置** | TASK-02 |
| **规格出处** | PRD §6.1, `knowledge-templates/kd-full.yaml` |

#### TASK-07：食谱池 + 花草茶池填充（24×9）

| 字段 | 内容 |
|------|------|
| **描述** | 填充24节气×9体质的食材池和花草茶配方池，JSON字段存储 |
| **验收标准** | 1. 216条食谱数据全部填充 2. 216条花草茶数据全部填充 3. 每条包含食材池JSON、天气修正权重JSON 4. 食谱池至少包含10种食材 5. 花草茶池至少包含3种配方 |
| **涉及文件** | `backend/src/config/recipe_pool.json`, `backend/src/config/herbal_tea_pool.json` |
| **前置** | TASK-02 |
| **规格出处** | PRD §6.1, 技术架构 §5.1 |

### Phase 3：API 服务（约1周）

#### TASK-08：后端 API 实现

| 字段 | 内容 |
|------|------|
| **描述** | 实现9个API端点（节气/方案/食谱/花草茶/体质/天气/倒计时/食材/健康检查） |
| **验收标准** | 1. 9个API全部实现 2. 每个API有请求验证 3. 响应格式与 PRD §7 一致 4. 错误处理完善（400/404/500） 5. 响应时间 ≤1秒 |
| **涉及文件** | `backend/src/routes/*.js`, `backend/src/services/*.js` |
| **前置** | TASK-05, TASK-06, TASK-07 |
| **规格出处** | PRD §7, 技术架构 §6 |

#### TASK-09：体质测评问卷实现

| 字段 | 内容 |
|------|------|
| **描述** | 实现体质测评API，输入8题答案，输出体质类型+得分，支持历史测评记录 |
| **验收标准** | 1. 8题是/否选择题 2. 加权评分算法（九种体质匹配） 3. 返回体质类型+每种体质得分 4. 支持重新测评 5. 历史测评记录可查 |
| **涉及文件** | `backend/src/services/ConstitutionService.js` |
| **前置** | TASK-05 |
| **规格出处** | PRD §4.1 功能2 |

### Phase 4：前端小程序（约3周）

#### TASK-10：首页 + 方案详情页 + 体质测评页

| 字段 | 内容 |
|------|------|
| **描述** | 实现三个核心页面：首页（节气卡片+方案卡片+花草茶卡片+体质测评入口+倒计时）、方案详情页（健康/起居/饮食/花草茶/节气知识/免责声明）、体质测评页（8题问卷+进度条） |
| **验收标准** | 1. 首页首屏加载 ≤2秒 2. 方案卡片可展开/收起 3. 花草茶卡片展示配方名称+食材+冲泡方法 4. 体质测评页每题一屏，进度条显示 5. 方案详情页包含所有维度 6. 字号≥16px，按钮≥44px，对比度≥4.5:1 7. 点击方案卡片进入详情页 8. 体质测评完成自动刷新首页方案 |
| **涉及文件** | `frontend/src/pages/index/`, `frontend/src/pages/plan/`, `frontend/src/pages/assess/`, `frontend/src/components/*.js` |
| **前置** | TASK-08, TASK-09 |
| **规格出处** | PRD §5.1, PRD §9 |
