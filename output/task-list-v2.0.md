## Phase 1：数据完善（阻塞性，可并行）

> 说明：当前仅有部分节气/体质的数据，需补齐至 24 节气 × 5+ 体质全覆盖。

### TASK-01：节气不变层数据完善

| 字段 | 内容 |
|------|------|
| **描述** | 补齐 24 节气完整不变层数据（气候/阴阳/脏腑/方向/脆弱点/三候） |
| **当前状态** | 部分节气已有（如小暑），需补齐剩余节气 |
| **验收标准** | 24 节气全部有数据，每条约 10 个字段，格式与 `solar_terms.json` 一致 |
| **涉及文件** | `backend/src/data/solar_terms.json` |
| **产出方式** | AI 辅助生成初稿 + 人工审核 |
| **前置** | 无 |

### TASK-02：城市气候数据扩充

| 字段 | 内容 |
|------|------|
| **描述** | 扩充城市气候数据至 100+ 城市（省会 + 热门旅游城市） |
| **当前状态** | 已有 `cities.json`，条目数待确认 |
| **验收标准** | ≥100 个城市，每个城市包含气候类型、冬夏温度、湿度范围、偏差方向 |
| **涉及文件** | `backend/src/data/cities.json` |
| **产出方式** | AI 辅助生成 + 人工抽查 |
| **前置** | 无 |

### TASK-03：体质适配表完善（9×24）

| 字段 | 内容 |
|------|------|
| **描述** | 补齐九种体质 × 24 节气适配数据（脆弱点/方向修正/程度修正/侧重点偏移） |
| **当前状态** | 已有 `constitutions-final.json`，条目数待确认 |
| **验收标准** | 216 条数据（9 体质 × 24 节气），每条包含 5 个字段 |
| **涉及文件** | `backend/src/data/constitutions-final.json` |
| **产出方式** | AI 辅助生成 + 人工审核 |
| **前置** | TASK-01（节气数据完整后） |

### TASK-04：食材池数据补齐（24×5+ 体质）

| 字段 | 内容 |
|------|------|
| **描述** | 为每个节气补齐至少 5 种体质的食材池（早餐/午餐/晚餐/汤品） |
| **当前状态** | 已有 `recipe_pool_xiaoshu.json`（小暑），需补齐其余节气 |
| **验收标准** | 每个节气至少覆盖 5 种体质（平和/气虚/阳虚/阴虚/痰湿），每种体质至少 3 道菜品 |
| **涉及文件** | `backend/src/data/recipe_pool_*.json`（按节气分文件） |
| **产出方式** | AI 辅助生成 + 人工审核 |
| **前置** | TASK-03（体质适配数据完整后） |

### TASK-05：花草茶配方池构建（24×5+ 体质）

| 字段 | 内容 |
|------|------|
| **描述** | 新建花草茶配方池，为每个节气至少覆盖 5 种体质 |
| **当前状态** | 仅有 `herbal_tea_pool_xiaoshu.json`（小暑示例），其余节气待补齐 |
| **验收标准** | 每个节气至少覆盖 5 种体质，每种体质至少 2 种花草茶配方，含食材清单、用量、冲泡方法 |
| **涉及文件** | `backend/src/data/herbal_tea_pool_*.json`（按节气分文件） |
| **产出方式** | AI 辅助生成 + 人工审核（标注禁忌体质） |
| **前置** | TASK-03（体质适配数据完整后） |
